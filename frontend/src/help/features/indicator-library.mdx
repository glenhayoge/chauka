# Indicator Library

The indicator library is a collection of reusable, pre-built indicators that can be shared across logframes within your organisation. Instead of creating every indicator from scratch, you can browse the library, find a relevant indicator, and add it to your logframe with one click.

## Why use the indicator library?

- **Save time** — pre-built indicators come with definitions, units of measure, data sources, and collection methods already filled in
- **Ensure consistency** — the same indicator definition is used across different logframes and projects
- **Follow standards** — library indicators can be aligned with sector frameworks and reporting standards
- **Reduce errors** — less manual entry means fewer mistakes in indicator definitions

## Browsing the library

1. In the **Design** tab, click on a result to expand it
2. Click **Use from Library** (next to the Add Indicator button)
3. The library dialog opens with a search and filter interface

### Searching and filtering

You can narrow down the library using:

- **Keyword search** — search by indicator name or description
- **Sector** — filter by sector (e.g., Health, Education, WASH, Agriculture)
- **Result level** — filter by the level the indicator is designed for (Impact, Outcome, Output)
- **Framework** — filter by reporting framework (e.g., SDG, donor-specific frameworks)

The library shows both **global indicators** (available to all organisations) and **organisation-specific indicators** created by your team.

## Previewing an indicator

Click on any indicator in the search results to see its full details:

- **Name** and **definition**
- **Unit of measure** (e.g., number, percentage, ratio)
- **Calculation method** (how the value is derived)
- **Data source** (where the data comes from)
- **Data collection method** (how data is gathered)
- **Reporting frequency** (how often it should be measured)
- **Disaggregation fields** (suggested breakdowns)
- **Framework and code** (alignment with standards)

## Adding an indicator from the library

1. Preview the indicator you want
2. Click **Use this indicator**
3. The indicator is copied into your logframe under the selected result
4. A default sub-indicator is created automatically
5. You can edit any of the indicator's details after adding it

The library indicator is copied, not linked — changes you make to your copy won't affect the library version, and vice versa.

## Creating library indicators

Organisation admins can add indicators to the library for their team to reuse:

- Define the indicator with full metadata (name, definition, unit, data source, etc.)
- Assign a sector and result level
- Optionally align with a framework and code
- Specify suggested disaggregation fields
- The indicator becomes available to all logframes in your organisation

## Best practices

- **Search before creating** — check the library before defining a new indicator from scratch
- **Use standard definitions** — align with your sector's accepted indicator definitions
- **Include disaggregation guidance** — note which disaggregation categories are recommended
- **Keep the library curated** — periodically review and update library indicators to ensure they remain relevant
- **Adapt after adding** — library indicators are a starting point; tailor the name, targets, and sub-indicators to your specific context
