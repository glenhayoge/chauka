# Organisations, Programs & Projects

Chauka uses a three-level hierarchy above your logframes to keep your work organised. Understanding this structure helps you set things up correctly from the start.

## The hierarchy

```
Organisation
  Programme
    Project
      Logframe
```

### Organisation

Your organisation is the top-level container. It represents your NGO, government agency, consultancy, or any entity managing development programmes.

- Each organisation has its own **members** with roles
- You can belong to multiple organisations
- Organisation admins manage all settings, members, and content

### Programme

Programmes group related projects under a common theme or funding stream. For example:

- "Health Programme"
- "Education & Livelihoods"
- "WASH Programme 2023-2028"

Programmes help you organise your work, but they don't have their own indicators or budgets — those live in logframes.

### Project

Projects are specific funded initiatives within a programme. Each project can have one or more logframes.

- Projects belong to a programme and an organisation
- Team members are assigned roles at the project level
- A project might have multiple logframes if it spans different phases or locations

## Setting up the structure

### Creating a programme

1. Go to your organisation dashboard or settings
2. Navigate to the **Programmes** section
3. Click **Add Programme**
4. Enter the programme name and description
5. Save

### Creating a project

1. Within a programme, click **Add Project**
2. Enter the project name, description, and key details
3. Assign team members and their roles (Lead, Data Collector, Viewer)
4. Save

### Creating a logframe

1. Within a project, click **Create Logframe**
2. Name your logframe
3. You'll be taken to the Design tab to start building your results framework

## Managing members

Team members are managed at two levels:

**Organisation level:**
- Admins can invite new members to the organisation
- Members can be given organisation-wide admin access

**Project level:**
- Project leads can assign roles to organisation members
- Roles determine what team members can do within that project's logframes

| Role | Create/Edit results | Enter data | View logframe |
|---|---|---|---|
| Admin | Yes | Yes | Yes |
| Lead | Yes | Yes | Yes |
| Data Collector | No | Yes | Yes |
| Viewer | No | No | Yes |

## Best practices

- **One organisation per entity** — don't create multiple organisations for the same NGO
- **Use programmes to group thematically** — not by year (use logframe naming for time periods)
- **One logframe per project phase** — if a project has distinct phases, create a logframe for each
- **Assign roles thoughtfully** — give people the minimum access they need
