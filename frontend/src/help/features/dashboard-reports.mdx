# Dashboard & Reports

The Dashboard gives you an at-a-glance view of your logframe's health. The Overview provides a comprehensive picture for reviews and reporting.

## The Dashboard tab

When you open a logframe, the Dashboard is the first thing you see. It provides:

### Key metrics
- Number of results, indicators, and activities
- Overall progress against targets
- Budget utilisation percentage

### Progress summary
- How many indicators are on track, at risk, or off track
- Activity completion status
- Budget spend rate

### Recent activity
- Latest data entries
- Recent status updates
- Team activity

The Dashboard is designed for quick check-ins. At a glance, you should be able to answer: "Is this project on track?"

## The Overview tab

The Overview provides a more detailed view of the entire logframe:

- Full results tree with status indicators
- Indicator summaries with target vs actual
- Assumption status and risk ratings

This view is useful for:
- **Programme reviews** — walk through the logframe with stakeholders
- **Board reports** — get the full picture in one view
- **Annual reviews** — assess overall programme performance

## Using status ratings

Chauka uses coloured status indicators throughout:

- Results show whether their indicators are collectively on track
- Indicators show individual performance against targets
- Assumptions show risk levels

These roll up from the bottom: if individual indicators are off track, the parent result will reflect that.

## Disaggregation analysis

The **Disaggregation** tab provides a dedicated view to break down your indicator data by categories such as Gender, Age Group, or District.

- Select a disaggregation category to view breakdowns
- Filter by reporting period to see data for a specific time window
- Toggle between total and average metrics
- View bar charts and summary tables showing values per group

This helps you identify disparities and ensure equitable reach. For example, you might see that a training programme reached 80% men but only 20% women, highlighting a gap to address.

See [Disaggregation Analysis](/help/features/disaggregation-analysis) for a full guide.

## Contribution analysis

The **Contribution** tab shows how output-level progress contributes to outcome-level results using weighted contribution factors.

- Each outcome displays an overall contribution score
- Child outputs show individual progress with their assigned weight
- Colour-coded progress bars indicate status: green (70%+), amber (40-69%), red (below 40%)

This helps managers understand which outputs are driving progress and which are lagging. See [Contribution Analysis](/help/features/contribution-analysis) for setup and usage.

## Organisation dashboard

Beyond individual logframes, the organisation dashboard (accessible from organisation settings) gives a portfolio view:

- Summary across all logframes in the organisation
- Programme-level aggregation
- Cross-project comparisons

## Printing and sharing

For offline sharing or donor reporting:

1. Use the **Print** view (available from the logframe menu) for a clean, printable version
2. Use **Excel export** for detailed data (see [Exporting Data](/help/features/exporting-data))

## Tips for effective monitoring

- **Check the dashboard weekly** — even a quick look helps catch issues early
- **Use the Overview for review meetings** — it gives everyone a shared picture
- **Focus on off-track indicators** — these need the most attention
- **Compare across periods** — trends matter more than individual data points
- **Share dashboards with stakeholders** — transparency builds trust
