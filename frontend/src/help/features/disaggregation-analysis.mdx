# Disaggregation Analysis

The Disaggregation tab lets you break down your indicator data by categories such as Gender, Age Group, or District. This reveals patterns and inequalities that aggregate numbers can hide.

## What is disaggregation?

Disaggregation means splitting your data into meaningful groups. For example, instead of just knowing "500 people were trained", you can see:

| Group | Count |
|---|---|
| Male | 310 |
| Female | 190 |
| **Total** | **500** |

This tells you that only 38% of trainees were women — information you'd miss looking at the total alone.

## Setting up disaggregation

Disaggregation analysis requires two things: categories and tagged sub-indicators.

### Step 1: Create disaggregation categories

1. Open your logframe
2. Go to the **Settings** tab
3. Find the **Disaggregation** section
4. Click **Add** and enter a category name (e.g., "Gender", "Age Group", "District")
5. Repeat for each category you want to track

Common categories include:

- **Gender** — Male, Female, Non-binary
- **Age Group** — Under 18, 18-35, 36-60, Over 60
- **Location** — Urban, Rural, or by district/region
- **Disability** — With disability, Without disability

### Step 2: Tag sub-indicators

1. Go to the **Design** tab
2. For each indicator that should be disaggregated, ensure it has sub-indicators (e.g., "Male", "Female")
3. Tag each sub-indicator with the appropriate disaggregation category and value

For example, for the indicator "Number of people trained":
- Sub-indicator "Male" is tagged with category **Gender**, value **Male**
- Sub-indicator "Female" is tagged with category **Gender**, value **Female**

### Step 3: Enter data

Enter actual values for each sub-indicator in the **Monitor** tab as you normally would. Chauka uses these values for the disaggregation analysis.

## Using the Disaggregation tab

Once categories are set up and data is entered, go to the **Disaggregation** tab to view the analysis.

### Filters

The tab provides three filters:

- **Category** — select which disaggregation category to view (e.g., Gender, Age Group)
- **Reporting period** — filter to a specific period, or view all periods combined
- **Metric** — choose between Total (sum of values) or Average (mean across indicators)

### Charts and tables

The analysis shows:

- A **bar chart** displaying values for each group within the selected category
- A **summary table** with total, count, and average for each group
- Visual comparison across groups to highlight disparities

## Reading the results

When interpreting disaggregation data:

- **Compare proportions** — are groups being reached equitably?
- **Track trends** — filter by different periods to see if gaps are narrowing or widening
- **Identify gaps** — which groups are underserved relative to their population share?
- **Inform decisions** — use the data to adjust your approach and resource allocation

## Tips

- **Set up categories early** — define your disaggregation categories before data collection begins
- **Be consistent** — use the same category values across all indicators (e.g., always "Male" not sometimes "M")
- **Start with what donors require** — most donors require at minimum gender-disaggregated data
- **Don't over-disaggregate** — too many categories create complexity without insight; focus on categories that drive decisions
