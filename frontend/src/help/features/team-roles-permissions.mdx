# Team Roles & Permissions

Chauka uses role-based access control to ensure team members can do what they need to — and nothing more. Roles are assigned at the project level and determine access to all logframes within that project.

## Role types

| Role | Description |
|---|---|
| **Organisation Admin** | Full control over the entire organisation: all programmes, projects, logframes, and members |
| **Project Lead** | Full edit access to logframes within their assigned projects |
| **Data Collector** | Can enter data against indicators in assigned projects (cannot change logframe structure) |
| **Viewer** | Read-only access to logframes in assigned projects |

## What each role can do

### Organisation Admin
- Create and manage programmes and projects
- Invite and remove organisation members
- Assign project roles
- Edit all logframes across the organisation
- Access organisation settings and dashboard
- Export data

### Project Lead
- Edit logframe structure (add/edit/delete results, indicators, activities)
- Enter data and set targets
- Manage activities and budgets
- View and add status updates
- Access all tabs within assigned logframes

### Data Collector
- Enter actual values against indicator targets
- Add status updates to activities
- View the logframe (all tabs)
- Cannot change the logframe structure (no adding/editing/deleting results or indicators)

### Viewer
- View all tabs of assigned logframes
- View dashboards and reports
- Cannot edit anything

## Assigning roles

### Inviting new members

1. Go to **Organisation Settings** > **Members**
2. Click **Invite Member**
3. Enter their email address
4. They'll receive an invitation email

### Assigning project roles

1. Go to the project's **People** section (or logframe People tab)
2. Find the team member
3. Assign their role (Lead, Data Collector, Viewer)
4. Save

### Changing roles

Organisation admins and project leads can change a member's role:

1. Go to the relevant People section
2. Click on the member's current role
3. Select the new role
4. Save

## Multi-project access

A team member can have different roles across different projects:

- **Lead** on "Water & Sanitation Project"
- **Data Collector** on "Health Programme Evaluation"
- **Viewer** on "Education Initiative"

This lets you give people exactly the access they need for each project.

## Best practices

- **Follow the principle of least privilege** — give people the minimum role they need
- **Use Data Collector for field staff** — they need to enter data but shouldn't change the logframe structure
- **Review roles periodically** — remove access when people leave or change responsibilities
- **Have at least two leads per project** — ensures continuity if someone is unavailable
- **Use Viewer for stakeholders** — they can see progress without risk of accidental changes
