# Entering Monitoring Data

The Monitor tab is where you record actual progress against your indicator targets. Regular data entry keeps your logframe current and your dashboards meaningful.

## Accessing the Monitor tab

1. Open your logframe
2. Click the **Monitor** tab
3. You'll see your results tree with indicators and reporting period columns

## Entering data

### Basic data entry

1. Find the indicator you want to update
2. Locate the current reporting period column
3. Click on the data entry cell
4. Enter the actual value
5. Save (or press Enter)

The cell will show your entered value alongside the target for easy comparison.

### Entering disaggregated data

If an indicator has sub-indicators, you'll enter data for each one:

1. Expand the indicator to see its sub-indicators
2. Enter the actual value for each sub-indicator in the current period
3. Chauka automatically calculates the aggregate total

For example, if "Number of people trained" has sub-indicators Male and Female:
- Male: 45
- Female: 62
- Total (auto-calculated): 107

## Understanding the Monitor view

The Monitor tab displays a grid:

- **Rows** are your results, indicators, and sub-indicators
- **Columns** are your reporting periods

Each cell can show:
- **Target** — what you planned to achieve
- **Actual** — what was actually achieved
- **Variance** — the difference between target and actual

## Data entry permissions

Not everyone can enter data:

| Role | Can enter data? |
|---|---|
| Admin | Yes |
| Lead | Yes |
| Data Collector | Yes |
| Viewer | No |

If you can't edit data entry cells, check your role with your project lead.

## Tips for effective data entry

- **Enter data promptly** — the sooner after the reporting period, the more accurate
- **Be consistent** — use the same units and counting methods across periods
- **Check disaggregation totals** — make sure sub-indicator values add up correctly
- **Add notes when needed** — if a value is unusual, document why (e.g., "Low due to flooding in March")
- **Review before finalising** — check entries against your source documents

## Reviewing progress

After entering data, check the **Dashboard** tab for:

- Visual progress charts
- Target vs actual comparisons
- Indicators that are off track
- Overall logframe health

If you have disaggregation categories set up, check the **Disaggregation** tab to see your data broken down by category (e.g., Gender, Age Group). This reveals patterns and inequalities that aggregate numbers can hide. See [Disaggregation Analysis](/help/features/disaggregation-analysis) for more.
