# Data Collection & Quality

Good monitoring depends on good data. This guide covers the fundamentals of collecting reliable data and ensuring its quality throughout the monitoring process.

## The data collection process

Data collection for M&E follows a structured process:

1. **Plan** — decide what data you need, from whom, how, and when
2. **Design tools** — create surveys, forms, templates, or observation guides
3. **Collect** — gather data in the field or from records
4. **Clean** — check for errors, inconsistencies, and missing values
5. **Enter** — input data into Chauka (Monitor tab)
6. **Verify** — cross-check entered data against source documents
7. **Analyse** — review trends, compare targets vs actuals

## Common data collection methods

| Method | Best for | Example |
|---|---|---|
| **Project records** | Output indicators | Training registers, distribution lists |
| **Household surveys** | Outcome indicators | Beneficiary surveys on wellbeing, income |
| **Key informant interviews** | Context, qualitative data | Interviews with community leaders |
| **Focus group discussions** | Understanding perceptions | Community feedback sessions |
| **Observation** | Service quality | Facility assessments, classroom observations |
| **Administrative data** | Population-level indicators | Government health records, school enrolment |
| **Digital data** | Real-time tracking | Mobile surveys (e.g., KoboToolbox), GPS tracking |

## Data quality dimensions

Good data meets five quality criteria:

### Validity
Does the data measure what it's supposed to measure?

- Ensure your indicator definition is clear and consistent
- Use validated survey instruments where possible
- Train data collectors on what counts and what doesn't

### Reliability
Would you get the same result if you measured again?

- Use standardised tools and procedures
- Train data collectors consistently
- Pilot-test your instruments

### Timeliness
Is the data collected and reported when it's needed?

- Set clear data collection schedules aligned with reporting periods
- Enter data promptly after collection
- Don't wait until the deadline to start

### Completeness
Is all the required data collected?

- Check for missing values before entering into Chauka
- Follow up on incomplete records
- Track response rates

### Integrity
Is the data protected from bias and manipulation?

- Use independent data verification
- Cross-check data from multiple sources
- Create clear data protocols

## Data collection planning

For each indicator in your logframe, document:

| Element | Description |
|---|---|
| **What** | Exactly what data you need (matches indicator definition) |
| **Who** | Who will collect it (staff, consultants, community volunteers) |
| **From whom** | The data source (beneficiaries, records, authorities) |
| **How** | The method (survey, observation, records review) |
| **When** | Collection frequency (monthly, quarterly) |
| **Where** | Geographic coverage (all sites, sample) |
| **Tool** | The instrument used (survey form, checklist) |

## Disaggregated data collection

If your indicators require disaggregation (by gender, age, location), plan for this from the start:

- Include demographic questions in your data collection tools
- Train data collectors to record disaggregation categories
- Set up **disaggregation categories** in Chauka's logframe settings (e.g., "Gender", "Age Group") before data entry begins
- Create sub-indicators and tag them with the appropriate category and value
- Check that disaggregated totals match aggregated figures

Once data is entered, you can use the **Disaggregation** tab to see automatic breakdowns with charts and summary tables. See [Disaggregation Analysis](/help/features/disaggregation-analysis) for details.

## Using Chauka for data management

### Setting up for data collection
1. Define your indicators and sub-indicators in the Design tab
2. Set reporting periods in Settings
3. Set targets for each indicator/period
4. Assign Data Collector roles to field staff

### Entering data
1. Data collectors enter actuals in the Monitor tab
2. Values are entered per indicator, per sub-indicator, per period
3. Chauka calculates aggregates automatically

### Quality checks
- Compare entered values against source documents
- Check that disaggregated sub-indicator values sum correctly
- Look for outliers — values that seem unusually high or low
- Verify that data was entered against the correct reporting period

## Common data quality issues

| Issue | Cause | Solution |
|---|---|---|
| Missing data | Incomplete collection | Follow-up, improve tools |
| Duplicate entries | Counting same item twice | Clear counting rules, unique IDs |
| Data entry errors | Typos, wrong fields | Double-check, verification |
| Outdated data | Late collection | Enforce schedules |
| Inconsistent definitions | Different interpretations | Clear indicator definitions, training |

## Best practices

- **Train your data collectors** — invest in proper training before each data collection round
- **Pilot your tools** — test instruments before full deployment
- **Enter data promptly** — the longer you wait, the more errors creep in
- **Verify a sample** — spot-check at least 10% of entries against source documents
- **Document your methods** — write down how data is collected so it's consistent across staff and periods
- **Keep source documents** — retain original records (paper or digital) for verification
