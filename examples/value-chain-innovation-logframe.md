# Converting FPDA's Annual Operation Plan to a Chauka Logframe

## Example: Value Chain Innovation Program

## Step 2: Build the Results Hierarchy

Enter these in the **Overview** tab, creating each level as a result:

```
Impact (Goal)
└── Strengthen partnerships to enhance efficiency and
    competitiveness in the value chain.

    Outcome 1
    └── Facilitate collaboration among stakeholders to
        reduce transaction costs.

    Outcome 2
    └── Promote innovative practices that improve supply
        chain efficiency.

        Output 1
        └── Improved Market-Oriented Extension Services
```

### How to create in Kashana:
1. Go to **Overview** tab
2. Click **"+ Click to add title"** → enter the Impact (Goal)
3. Expand the Impact → click **"+ Click to add child result"** → enter Outcome 1
4. Repeat for Outcome 2
5. Expand Outcome 2 → add Output 1

Budget totals automatically roll up:
- **Activity total** = sum of budget lines
- **Output total** = sum of all activities under it
- **Outcome total** = sum of all outputs
- **Program total** = sum of all outcomes

## Step 7: Monitor Progress

Go to the **Monitor** tab to track actuals against targets.

### What you'll see:
- **Baseline** column showing 2024 achieved values
- **Next Target** column showing current period targets
- **Measurement columns** for entering actual values as data comes in
- **Rating** to mark each indicator as On Track / Caution / Off Track

### How to enter actuals:
1. Click **"+ Add measurement column"** (creates a dated column)
2. Click any cell to enter the actual value
3. Click **"Set rating"** to assess progress

### Example tracking:

| Sub-indicator | Baseline (2024) | Target (2025) | Q1 Actual | Q2 Actual | Rating |
|--------------|----------------|---------------|-----------|-----------|--------|
| VC partners trained | 44 | 50 | 12 | 28 | On Track |
| Contact farmers trained | 28 | 30 | 5 | 14 | Caution |
| Supply chain players | 12 | 15 | 3 | 7 | On Track |

## Step 8: Check Workload

Go to the **Workload** tab to see staff allocation across activities.

### Staff Table view
Shows each person's monthly allocation percentage. Red cells flag over-allocation when a staff member is assigned >100% in any month.

### Timeline view
Gantt-style chart showing overlapping activity assignments per person.

### By Role view
Summarises demand by role (Extension Officers, Trainers, etc.) across months.

## Step 9: Review Budget Utilisation

Go to the **Budget** tab for a financial overview.

### What you'll see:
- **Total Budget**: PGK 400,000
- **Total Spent**: sum of recorded expenses
- **Remaining**: budget minus spent
- **Utilisation %**: with color-coded progress bars
- Per-activity breakdown with expandable expense details

### How to record expenses:
1. Go to Budget tab
2. Expand an activity → expand a budget line
3. Click **"+ Add expense"**
4. Enter date, amount, and description

## What Chauka Adds Beyond the Original Plan

| Feature | Original Plan | Chauka |
|---------|--------------|---------|
| **Structure** | Flat table | Hierarchical: Goal → Outcome → Output → Activity |
| **Tracking frequency** | Annual only | Quarterly (or monthly/semi-annual) |
| **Progress monitoring** | None | Actuals vs targets with traffic-light ratings |
| **Budget tracking** | Single line total | Categorised budget lines with expense tracking |
| **Risk management** | Not included | Risk ratings and assumptions per result |
| **Resource planning** | Not included | Staff, equipment, and partner assignments |
| **Workload visibility** | Not included | Over-allocation detection and timeline view |
| **Collaboration** | Paper-based | Multi-user with roles (admin, lead, collector, viewer) |
| **Reporting** | Manual | Automated quarterly/annual exports |
| **Status updates** | Not included | Narrative progress notes per activity |

## Tips for Data Entry

1. **Start with the hierarchy** — get the Goal, Outcomes, and Outputs right before adding activities
2. **One indicator per KPI** — each row in the original plan becomes one indicator in Chauka
3. **Set baselines first** — enter 2024 achieved values before setting 2025 targets
4. **Add assumptions** — what must be true for each output to contribute to its outcome
5. **Use budget categories** — makes it easier to track spending patterns across activities
6. **Assign resources early** — helps identify staffing conflicts before they happen
7. **Enter actuals quarterly** — don't wait until year-end to track progress
8. **Use status updates** — record narrative context that numbers alone can't capture
